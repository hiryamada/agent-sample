# 技術書の出版計画（実行用）

## 前提条件
- ジャンル: 技術書
- 想定読者: 特定分野の実務者向け
- 出版形態: 紙の本のみ
- 目標時期: 2027年2月
- 予算: 制約なし
- 原稿状況: 構想のみ
- 推奨ルート: 自費出版寄り（スケジュール確実性を優先）

## 企画方針
- 読者の業務課題を解決することを主目的にする
- 実務で再現できる手順、図解、サンプル、チェックリストを重視する
- 1冊で完結するが、必要なら拡張可能な構成にする

## 章立ての基本案
1. 序章: なぜこの技術が必要か
2. 基礎概念の整理
3. 実践環境の構築
4. コア手順・実装
5. よくある失敗と対策
6. 応用パターン
7. 運用・保守・改善
8. 付録: サンプル、用語集、参考情報

## 制作フロー
1. 企画確定
2. 目次確定
3. 章ごとの執筆
4. 技術レビュー
5. 編集・推敲
6. 図版・表組み整備
7. 校正
8. 入稿
9. 印刷・製本
10. 配本・発売準備

## 月次スケジュール（2026年7月〜2027年2月）
- 2026年7月: 企画テーマの絞り込み、読者課題整理、目次の大枠作成
- 2026年8月: 目次確定、章ごとの執筆計画作成、必要素材の洗い出し
- 2026年9月: 第1〜2章の初稿
- 2026年10月: 第3〜4章の初稿
- 2026年11月: 第5〜6章の初稿
- 2026年12月: 第7〜8章の初稿、全体の通読
- 2027年1月: 技術レビュー、編集、推敲、図版整理、校正
- 2027年2月: 最終校正、入稿、印刷進行、発売準備

## 週次の進め方（目安）
- 毎週: 1章分または1節分の執筆進捗を出す
- 毎週末: レビューと修正点の整理
- 月末: その月の章を締め切る

## チェックポイント
- 目次が読者課題に合っているか
- 章ごとに実務再現性があるか
- 図やサンプルが不足していないか
- 技術的な正確性が担保されているか
- 2027年2月刊行に対して遅延がないか

## 次のアクション
- 具体的なテーマを1文で定義する
- 想定読者の職種・レベルをさらに明確にする
- 目次案を詳細化する
